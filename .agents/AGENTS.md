# General Coding Rules

Follow these rules for all future code modifications in this project:

- **Record New Rules:** When the user provides a general rule in their request, proactively add it to this AGENTS.md document without them having to explicitly ask.

- **Naming Convention:** Use `camelCase` consistently across the codebase.
- **DRY (Don't Repeat Yourself):** Avoid code duplication. Extract repetitive logic into helper functions, configuration objects, or classes.
- **Object-Oriented Design:** Use Classes to encapsulate state and related logic whenever relevant, rather than relying on global variables.
- **Clean Code:** Delete unused variables, empty functions, and dead code immediately. Do not leave them behind in the codebase.
- **Professional Formatting:** Keep the codebase strictly organized, professional, and visually concise.
- **Documentation over Comments:** Do not write unnecessary or "stupid" inline comments explaining basic code. Instead, write clean, self-documenting code. Only use professional documentation blocks when a piece of logic (like complex math) truly requires it.
